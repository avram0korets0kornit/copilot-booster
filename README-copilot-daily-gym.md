## Why small daily changes?

Big refactorings are intimidating and risky. Small daily improvements are:
- **Manageable**: 15 minutes a day beats weekend marathons
- **Safe**: Small changes are easier to review and revert
- **Cumulative**: Tiny improvements compound over time
- **Sustainable**: Building a habit is better than burning out

Software quality isn't built in a day. It's built in many small days.

## How to Use This Repo

1. **Clone it**:
   ```bash
   git clone https://github.com/your-username/copilot-daily-gym.git
   cd copilot-daily-gym
   ```

2. **Run the tests**:
   ```bash
   pytest tests/
   ```

3. **Pick something to improve**:
   - Find a TODO comment
   - Add a missing docstring
   - Refactor a verbose function
   - Add a test case

4. **Make the change** with GitHub Copilot's help

5. **Commit it** and feel good about yourself

## Daily Practice Ideas

- **Thursday**: Add test cases for edge cases
- **Friday**: Improve variable naming
